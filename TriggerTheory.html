Trigger Theory Questions

What is difference between workflow process builder and trigger?

workflow and process builder work only after action,and trigger is work on before and after action.

Q)WHAT IS APEX TRANSACTION

Q)What is trigger?

trigger it is piece of code that execute before or after record is updated/inserted/deleted from the database.

Different ways of trigger?

  • using standard navigation
  • using devoloper consoleq
  • using eclipse ide

Different events of trigger?

  • before insert=we have to executing logic before inserting record into the object.eg.validation rule
  • after insert=by using this we can fire action after the inserting the record into obj,ex-email alert
  • before update=we have to invoke the action before updating record into the object
  • after updating=by using this we can fire action after the updating the record into obj,ex-email alert,sending email notification
  • before delete=we can fire the action before deleting record from object.eg-removing all child records
  • after delete=we can fire the action after deleting record from object.ex-rollup summary
  • after undelete=we can fire the action after restoring the records back to the object.

Different context variable

  • is executing:it return true if curreent executing apex code is trigger.
  • is update-it return true,if trigger was fired due to the update operation.
  • is insert-it return true,if trigger was fired due to the insert operation.
  • is before-it return true,if trigger was fired before the recored is saved.
  • is after-it return true,if trigger was fired after the record is saved.
  • is new-return list of new version of sobject records.
  • is old-return list of old version of sobject records.
  • is new map-return a map of new version of sobject record.
  • is old map-return map of old version of sobject record
  • is size-it return intger (total no of records due to trigger invocation for the both old and new);

Best practices of trigger

  • one object one trigger
  • always use helper and handler method inside the trigger
  • trigger should be logicless
  • never write query inside for loop
  • queries should be streaming (filtered)
  • avoid hard coded id,s.
  • proparly use of future method inside trigger.
  • trigger should be bulkified.

Execusion flow of Trigger/saleforce/apex class

  • load the data from database
  • check for the system validation
  • before trigger
  • custom validation
  • after trigger
  • workflow and process processbuilder
  • workflow and process builder any change before and after the trigger
  • rollup summary
  • sharing and shecurity
  • post commit opertion like email alert
